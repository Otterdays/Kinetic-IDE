// Architecture Blueprint v1.0

Android Tablet IDE
Kotlin-Native Stack

A production-grade architecture for an AI-agent-driven IDE on Android tablets. Four layers. Each owns its domain completely.

// Layer Architecture
L1 KOTLIN UI & Interaction Layer Jetpack Compose + Material You
Everything the user touches. Code editor canvas, file tree, agent chat panel, terminal pane, split layouts. Drawn entirely in Compose — no XML, no legacy Views unless needed for edge cases. Handles stylus input, keyboard shortcuts, multi-window, drag-and-drop natively.
Jetpack Compose Material You Canvas API WindowManager StylusMotionEvent Compose Animations LazyColumn / LazyTree Custom DrawScope
↓ State / ViewModel / Coroutines ↓
L2 KOTLIN Agent Orchestration Layer Coroutines + Anthropic API + LSP Client
The intelligence hub. Manages the Anthropic API stream, tool call dispatch, LSP protocol client (language server communication), conversation history, and all async I/O. Every agent tool call routes through here to the layers below. Kotlin coroutines make streaming responses and parallel tool execution clean and readable.
Kotlin Coroutines Flow / StateFlow OkHttp SSE streaming Anthropic Messages API LSP4J client Tool call router Conversation state ViewModel + Hilt DI
↓ JNI Bridge / Unix Sockets ↓
L3 RUST Performance Core Layer Rust NDK + tree-sitter + ropey
The engine. Syntax parsing via tree-sitter (same parser powering Neovim, Zed, GitHub). Rope data structure for efficient large-file editing. Fuzzy file search, diff/patch engine, terminal emulator core (VT100/ANSI). Called from Kotlin via JNI. This is optional in Phase 1 — replace with Kotlin equivalents initially, migrate hot paths to Rust later.
tree-sitter ropey (rope buffer) similar (diff) ignore (gitignore) alacritty_terminal nucleo (fuzzy search) JNI via jni-rs tokio async runtime
↓ Android NDK / POSIX / Termux ABI ↓
L4 NDK / C System & Filesystem Layer Android NDK + Termux bootstrap + Linux ABI
Raw access to the device. Filesystem R/W via Android Storage APIs + Termux shared storage. Process spawning (compile, run, git). Unix domain sockets for IPC with language servers. File watching via inotify. This layer is the bridge between your IDE and the actual Linux environment Termux provides.
Android Storage API ProcessBuilder / NDK fork Unix domain sockets inotify file watcher Termux:API bridge SAF (Storage Access Framework) WorkManager (bg tasks)

// Key Modules
Kotlin · UI

CodeEditorPane

Custom Compose Canvas with line numbers, syntax token coloring, cursor blink, selection highlight, gutter for breakpoints/errors.

Kotlin · UI

AgentChatPanel

Streaming markdown renderer. Tool call visualizer shows the agent's live actions. Resizable split-pane with the editor.

Kotlin · Agent

AnthropicClient

OkHttp SSE client. Parses streaming JSON events. Emits to a Kotlin Flow that the UI collects. Full tool use support.

Kotlin · Agent

ToolRouter

Maps tool names from the API response to actual Android operations. Routes to Rust layer or NDK layer based on tool type.

Rust · Core

SyntaxEngine

tree-sitter parser pool. Incremental re-parse on edit. Returns token spans to Kotlin for rendering. Handles 50k+ line files smoothly.

Rust · Core

DocumentBuffer

Rope-based text buffer. O(log n) insert/delete. Undo history. Serializes to string on demand for API calls or file writes.

Kotlin · LSP

LSPManager

Spawns language server processes (clangd, kotlin-ls, pyright). Speaks LSP over stdio. Feeds completions, diagnostics, hover docs to UI.

NDK · System

ProcessHost

Spawns compile/run/git processes. Streams stdout/stderr back as a Flow. Handles signals, timeouts, working directory.

// Code Sketch — Agent Stream Handler
AgentViewModel.kt Kotlin 
@HiltViewModel
class AgentViewModel @Inject constructor(
    private val client: AnthropicClient,
    private val toolRouter: ToolRouter,
    private val fileSystem: FileSystemRepository
) : ViewModel() {

    val agentState = MutableStateFlow(AgentState.Idle)
    val messages = MutableStateFlow<List<Message>>(emptyList())

    fun sendMessage(userText: String) {
        viewModelScope.launch {
            agentState.emit(AgentState.Streaming)

            // Stream tokens directly into UI state
            client.streamMessages(
                messages = messages.value,
                tools = toolRouter.tools
            ).collect { event ->
                when (event) {
                    is StreamEvent.TextDelta -> appendToken(event.text)
                    is StreamEvent.ToolUse  -> handleToolCall(event)
                    is StreamEvent.Done     -> agentState.emit(AgentState.Idle)
                }
            }
        }
    }

    private suspend fun handleToolCall(event: StreamEvent.ToolUse) {
        val result = toolRouter.dispatch(event.name, event.input)
        // Result flows back into next API call as tool_result
        client.sendToolResult(event.id, result)
    }
}
CodeEditorCanvas.kt Kotlin · Compose 
@Composable
fun CodeEditorCanvas(
    buffer: DocumentBuffer,
    tokens: List<SyntaxToken>,
    modifier: Modifier = Modifier
) {
    val scrollState = rememberScrollState()
    val density = LocalDensity.current

    Canvas(modifier = modifier.verticalScroll(scrollState)) {
        // Draw gutter
        drawRect(color = gutterColor, size = Size(gutterWidth, size.height))

        tokens.forEachIndexed { index, token ->
            drawText(
                textMeasurer = textMeasurer,
                text = token.text,
                style = TextStyle(color = token.syntaxColor, fontSize = 14.sp),
                topLeft = Offset(token.x, token.y)
            )
        }

        // Cursor blink handled by Animatable
        if (cursorVisible) {
            drawLine(color = cursorColor, start = cursorStart, end = cursorEnd, strokeWidth = 2f)
        }
    }
}
